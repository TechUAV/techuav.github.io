Прошивка MILBETA v1.48
Руководство пользователя
Функциональность
Прошивка основана на Betaflight 4.4.3, 4.5.1 или 4.6.0 и добавляет следующие функции:
• Постановка на запуск после 40 минут ожидания
• Снятие блокировки с охраны (кроме дистанционного)
• osd_aux_offset – смещение для дополнительного значения на экранном меню
• Вывод данных RSSI с двух антенн в экранное меню
• Поддержка системы NORDA Underdog Proof System
• Количество полос в vtxtable увеличено до 16 (v1.24)
• Сняты ограничения по частоте ( v1.20)
• Пеленгация средств радиоэлектронной борьбы на ОСД
• Пользовательский текст для osd_aux_channel
• Маскировка OSD для сокрытия положения пилота
• Уровни каналов управления в виде иконок
• Отображение статистики РЭБи LQ по частотам на ОСД
• Переключение VTX-каналов в режиме рабочей станции
• Расширение FAILSAFE
Полная история изменений в конце документа.
Распространение
Несите ответственность за распространение этой информации:
• Описание прошивки должно распространяться только в группах с верифицированными
участниками
• Передавайте скомпилированное встроенное ПО только тем, кого вы знаете или можете
проверить
• направлять заинтересованные лица к разработчику (@dankzb.03 в сигнале) для получения новых
версий и в качестве дополнительного способа проверки

Активация
Активация не требуется при использовании MILBETA в сочетании с приемником на MILELRS.
В остальных случаях необходимо активировать каждый полетный контроллер на MILBETA для того,
чтобы дополнительный функционал работал должным образом.
1. В Betaflight Configurator на вкладке CLI выполните команду "get mb_id"
2. Перейдите в milpilots.com, вкладка MILBETA, сгенерируйте ключ на основе полученного mb_id
3. В Конфигураторе Betaflight на вкладке CLI выполните команду "set mb_lock = XXX.XXX.XXX.XX", где
XXX.XXX.XXX.XX MB_LOCK с сайта
4. Виконайте команду "save"


Статистика OSD EW и LQ
Модуль RX с прошивкой MILELRS 2.40+ отправляет на полетный контроллер статистику по потере
пакетов по частотам и мощности сигнала РЭБ.
Для того, чтобы показать эту статистику на OSD, нужно установить прошивку MILBETA вместо стандартной
Бетафлайт.
Статистика включает в себя качество связи (LQ:0-99) и уровень сигнала радиоэлектронной борьбы (EW:0-
99) в 12 частотных диапазонах. Сигнал 0 соответствует -128 дБм, 99 - 0 дБм.

Настройки экранного меню
Для того, чтобы отобразить статистику по OSD, необходимо добавить на экран в Конфигураторе Betaflight
поле ESC RPM:



Формат отображения задается в поле Имя пилота (можно оставить пустым):

Каждая из букв имени пилота может иметь следующие значения:
Буква 1: Направление текста
V - вертикальная
H - горизонтальная

Буква 2: Формат значений EW.
Возможные значения:
A - Стрелка: Значение в форме стрелки
B - Battery: Значение в форме батареи, разряженная батарея - сигнал 0, полная - 99
b - battery: Перевернутое значение в виде батарейки, разряженная батарея - сигнал 99,
полная - 0 N - Number: Числовое значение в диапазоне от 0 до 99
R - Raw: Числовое значение в дБм в диапазоне от -128 до
0 T: Иконка (если загружен шрифт)
n - число: Числовое значение в диапазоне от 0 до 9

Буква 3: Формат значения качества связи (LQ).
Возможные значения:
B - Battery: Значение в форме батареи, разочарованная батарея - качество
0, полная - 99 N - Число: Числовое значение в диапазоне от 0 до 99
n - number: Числовое значение в диапазоне от 0 до
9 T : Иконка (если шрифт загружен)

Буква 4: Количество частотных диапазонов
Возможные значения:
* - 12 полос

6 - 6 полос
4 - 4 полосы
3 - 3 полосы
2 - 2 полосы

Буква 5: Минимальное значение сигнала РЭБ
Возможные значения:
от 0 до 9
0 - сигнал 0
9 - Сигнал 90

Буква 6: Максимальное значение сигнала РЭБ
Возможные значения:
от 0 до 9
0 - Сигнал 9
9 - Сигнал 99

Минимальное и максимальное значения РЭБ используются для того, чтобы сделать использование
иконок более наглядным. Например, если реальный уровень сигнала находится в диапазоне от 25 до
66, то последние две буквы можно установить на 26, то есть на иконке может отображаться сигнал от 20
до 69.

Рекомендуемая строка для имени пилота — «HTT*16».

Для того, чтобы иметь возможность использовать конкретные символы, которые занимают меньше
места на экране, нужно загрузить шрифт (font) в полетный контроллер.

Пеленгация средств радиоэлектронной борьбы на ОСД
MILELRS v2.52+ отправляет на полетный контроллер информацию о сигнале радиоэлектронной борьбы с
двух антенн приемника RX. Дополнительные сведения о пеленгации см. в файле EW_SCANNER.pdf
Настройки экранного меню:

Информация отображается в экранном меню в следующем виде:
Первая цифра – это средний уровень сигнала РЭБ для двух антенн в диапазоне от 0 до -128 дБм.
Последнее число – это разница в сигналах между первой и второй
антеннами. Две колонки показывают пилоту, с какой стороны сигнал
РЭБ сильнее.
Символ «>» указывает на то, что нужно повернуть дрон вправо, в сторону более сильного сигнала.

Расширение FAILSAFE
Betaflight ограничивает продолжительность этапа 1 до 20 секунд.
В MILBETA значение в интерфейсе конфигуратора по умолчанию умножается на 10. То есть 10 секунд в
интерфейсе конфигуратора фактически означает 100 секунд.

Таким же образом умножается время на посадку. 5 в примере означает 50 секунд.

Коэффициент удлинения можно изменить с помощью
интерфейса командной строки (v1.47+): установите
mb_delay_factor = 10
устанавливаем mb_landing_factor = 10

Настройка osd_aux_channel
В некоторых случаях, например, если в качестве платы инициации используется полетный контроллер,
возникает необходимость вывода состояния канала управления в экранное меню.
1. Активируйте соответствующий элемент на экранном меню

2. Укажите канал управления, связанный с этим элементом:
набор osd_aux_channel = 3

В этом примере используется канал 7 (AUX3).

3. Для того, чтобы отобразить текст с иконками вместо значений цифрового канала, выполните
следующие команды:

набор osd_aux_symbol = 77
set osd_aux_text1 = xCFWAIT
set osd_aux_text2 = xCEREADY
set osd_aux_text3 = xCD**BOOM**
сохранить

Иконка закодирована тремя символами, начинающимися с буквы 'x' (например, xCF). Коды иконок
можно найти в разделе с таблицей символов.
В экранном меню строка «xCFWAIT» отображается как


osd_aux_offset
Если канал управления закреплен за сервоприводом, то удобно отображать положение сервопривода в
градусах. Для этого можно воспользоваться стандартными osd_aux_scale настройками. Если положение
сервопривода может иметь отрицательные значения, то вы можете использовать нестандартный
параметр osd_aux_offset для отображения реального угла поворота на экранном меню.
Команда в CLI:
устанавливаем osd_aux_offset = -90

• После перезагрузки — применение пользовательских значений по умолчанию

1. Выполните команду dump all

2. Сохраните настройки в файл


3. Если текущая версия отличается от новой, то установите стандартную прошивку нужной версии
